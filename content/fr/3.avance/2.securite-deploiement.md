## Recommandations de sécurité

::callout{icon="i-lucide-shield" color="green"}
TradeJourney stocke toutes vos données localement. Aucune donnée n'est envoyée vers des serveurs externes.
::

1. **Ne pas exposer sur Internet** sans reverse proxy (nginx, Caddy, Traefik)
2. **Utiliser HTTPS** en production
3. **Sauvegarder régulièrement** vos données
4. **Mettre à jour** régulièrement l'application
5. **Protéger le fichier `.env`** — ne jamais le commiter dans git

## Déploiement en production

### Avec un reverse proxy nginx

```nginx [nginx.conf]
server {
    listen 443 ssl;
    server_name tradejourney.yourdomain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### Étapes recommandées

1. Utilisez un reverse proxy (nginx, Traefik, Caddy)
2. Configurez SSL/TLS (Let's Encrypt recommandé)
3. Limitez l'accès par IP si possible
4. Activez les backups automatiques
5. Surveillez les logs

## Dépannage

### L'application ne démarre pas

```bash [Terminal]
# Vérifier les logs Docker
docker-compose logs -f

# Vérifier si le port est déjà utilisé
lsof -i :3000
```

### Erreur de base de données

```bash [Terminal]
# Vérifier les permissions
chmod -R 755 data/

# Recréer la base si nécessaire (sera recrée au prochain démarrage)
rm data/user_X_data/database/database.db
```

### Performances lentes

```bash [Terminal]
# Optimiser la base de données SQLite
sqlite3 data/user_X_data/database/database.db "VACUUM;"
```

Pour augmenter les ressources Docker, ajoutez dans `docker-compose.yml` :

```yaml [docker-compose.yml]
deploy:
  resources:
    limits:
      memory: 2G
```
