## Filtres avancés

Les filtres sont disponibles sur toutes les pages de TradeJourney.

### Critères disponibles

| Filtre      | Description                           |
| ----------- | ------------------------------------- |
| **Symbole** | Filtrer par instrument (ex: `EURUSD`) |
| **Type**    | Buy ou Sell                           |
| **Date**    | Période personnalisée                 |
| **Tags**    | Un ou plusieurs tags                  |
| **Compte**  | Un ou plusieurs comptes               |
| **PnL**     | Valeur min et/ou max                  |
| **Durée**   | Trades courts ou longs                |

### Opérateurs disponibles

| Opérateur | Signification     |
| --------- | ----------------- |
| `=`       | Égal              |
| `!=`      | Différent         |
| `>`       | Supérieur         |
| `>=`      | Supérieur ou égal |
| `<`       | Inférieur         |
| `<=`      | Inférieur ou égal |
| `in`      | Dans la liste     |

### Sauvegarde des filtres

Les filtres actifs sont automatiquement sauvegardés dans votre session. Ils sont restaurés à la prochaine visite.

## Export de données

### Exporter en CSV

1. Appliquez vos filtres souhaités
2. Cliquez sur **Export** → **CSV**
3. Le fichier est téléchargé automatiquement

### Exporter en JSON

Pour une sauvegarde complète :

1. Allez dans **Paramètres** → **Export**
2. Sélectionnez **JSON**
3. Téléchargez le fichier

### Backup de la base de données

La base SQLite peut aussi être sauvegardée directement :

```bash [Terminal]
cp data/your-database.db backup/your-database-backup.db
```

Pour automatiser les backups :

```bash [backup.sh]
#!/bin/bash
BACKUP_DIR="backup/$(date +%Y%m%d-%H%M%S)"
mkdir -p $BACKUP_DIR
cp -r data/ $BACKUP_DIR/
echo "Backup créé dans $BACKUP_DIR"
```

### Restaurer une sauvegarde

```bash [Terminal]
# Arrêter l'application
docker-compose down

# Restaurer les données
rm -rf data/
cp -r backup/data-20240315/ data/

# Redémarrer
docker-compose up -d
```
